# Security policy

## Reporting a vulnerability

Please do **not** report security vulnerabilities in public issues or discussions.

Use GitHub's private vulnerability reporting for this repository:

- Go to the Security tab in this repository
- Use "Report a vulnerability"

If private reporting is unavailable for any reason, open a GitHub Discussion and ask for a private contact path without posting vulnerability details.

## What to include

Please include:

- Affected versions
- Impact and attack scenario
- Reproduction steps or proof of concept
- Suggested mitigation if known

## Response expectations

- Initial acknowledgement target: within 7 days
- Triage and impact assessment target: within 14 days
- Fix timeline depends on severity, complexity, and release timing

## Scope

This policy applies to the ChoreOps repository and its maintained releases.
